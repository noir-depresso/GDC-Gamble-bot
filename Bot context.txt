# Discord Card Turn-Based Bot (C#) → Unity Transfer

## Project goal

Build a **text-based Discord bot** that runs a **turn-based card combat game** (player vs enemy) with:

* **HP combat** + **Money / Basic Income (BI)** economy loop
* Cards that are **data + effects**, with reusable effect templates
* Code structured so the **game engine is pure C#** and can later be reused in **Unity** (swap Discord I/O for Unity UI)

# Discord bot token / env

Token stored as **User environment variable**:

* Key: `BOT_TOKEN`
* Verified via:

  * `$env:BOT_TOKEN` (was empty in current PowerShell session)
  * `[System.Environment]::GetEnvironmentVariable("BOT_TOKEN","User")` (returned correctly)

Important note:

* If bot prints `BOT_TOKEN exists? false`, it’s usually because:

  * the program reads **Process env**, but PowerShell session wasn’t restarted after `setx`, or
  * code reads `EnvironmentVariableTarget.Process` implicitly (default `GetEnvironmentVariable("BOT_TOKEN")` uses process), so user-level token may not be present in that session.
* Fix: restart terminal OR set for session:

  * `$env:BOT_TOKEN="..."`

# Commands supported (current bot design)

These are the intended Discord text commands (prefix `!`):

* `!help`
  Shows command list
* `!newgame`
  Creates a new game session for the channel (usually 1 game per channel for now)
* `!status`
  Shows player/enemy HP, money, BI, income multiplier, turn number, key stacks
* `!hand`
  Shows current hand with indices and each card’s **final cost** (after discounts)
* `!play <index>`
  Plays a card from the hand by index
* `!end`
  Ends player turn → enemy attacks → end-of-round payout → investment tick → duration tick → draw 1 → next turn

(If not all are currently wired in Discord.Bot yet, that’s the intended interface and structure.)

# Core game rules / loop (engine logic)

## Entities

* `Combatant`

  * `Attack`
  * `MaxHealth`
  * `CurrentHealth`
  * `TakeDamage(amount)` and `Heal(amount)`
* `GameState`

  * `BasicIncome` (BI)
  * `Money`
  * `IncomeMultiplier` (affects next payout only; resets after payout)
  * Status dictionary: `Dictionary<string, StatusInstance>`

    * stacks + duration (-1 = permanent)
  * Helpers:

    * `GetStacks(id)`
    * `AddStacks(id, add, durationTurns)`
    * `RemoveStatus(id)`
    * `TickDurations()`
    * `IncomeThisRound()` = round(BI * IncomeMultiplier)
    * `Rand01()` / `RandRange()`

## Turn structure

### Player action

* Player can play cards as long as they can afford them (Money-based cost)
* Playing a card:

  * compute `FinalCost(card)` using modifiers (currently Buy Low)
  * pay cost
  * remove from hand → discard
  * run all card effects that trigger on `OnPlay`
  * check win (enemy dead)

### End turn sequence

When `EndTurn()` is called:

1. Enemy attacks (base `Enemy.Attack`)
2. Trigger defense pipeline:

   * create `EffectContext` with `PendingDamage = incoming`
   * `_runner.FireTrigger(OnBeforeTakeDamage, ctx)` may reduce/modify damage
3. Apply damage to player
4. Check loss (player dead)
5. End of round economy:

   * income payout: `Money += IncomeThisRound()`
   * reset `IncomeMultiplier = 1`
6. Investment tick:

   * `_runner.FireTrigger(OnRoundEnd, ctx)`
     (pays out things like Bank/SellHigh/Stocks drift, etc)
7. Turn-end triggers (if implemented): `_runner.FireTrigger(OnTurnEnd, ctx)`
8. Tick durations: `State.TickDurations()`
9. Start next turn:

   * `TurnNumber++`
   * draw 1 card

## Deck/hand/discard

* Deck implemented via:

  * `_deck: Queue<CardDef>`
  * `_discard: List<CardDef>`
  * `Hand: List<CardDef>`
* Draw rules:

  * Start game draw 5
  * Each new turn draw 1
  * When deck empty: shuffle discard back into deck

# Current effect templates (known)

In `Game.Core.Effects.Impl` you now have:

* `AddStacksEffect(id, addStacks, durationTurns)`
* `AddMoneyFromBIEffect(percent)`
* `AddBasicIncomeEffect(amount)`

* “passive payout” is handled by `EffectRunner` reading stacks on triggers like `OnRoundEnd`

# Cost modifier logic (Buy Low)

Final cost formula in `GameEngine.FinalCost(card)`:

* `stacks = min(State.GetStacks(BUY_LOW), 2)`
* multiplier = `1 - 0.10 * stacks`
* finalCost = round(baseCost * multiplier), clamped >= 0

This makes Buy Low a **rule modifier** rather than a payout effect.

# What the next AI should do next

## Immediate next step: “Phase 1” features (timing-heavy cards)

Add cards/effects that require triggers beyond OnPlay/OnRoundEnd:

1. **Hedging**

   * OnPlay: `IncomeMultiplier = 0.5` for next payout
   * Add a 1-turn “shield” stack/status
   * OnBeforeTakeDamage: if shield active → reduce PendingDamage by 50%
   * duration ticks remove shield after 1 turn

2. **Enchantment**

   * OnPlay: add status “ENCHANT_NEXT” duration 1
   * OnBeforeDealDamage (new trigger) or handled inside DealDamage pipeline:

     * if ENCHANT_NEXT active, multiply outgoing damage by 1.5 then consume

3. **Discredit**

   * OnPlay: add status “DISCREDIT_READY” duration 1
   * OnDamageTaken: if active, reflect 50% to attacker and consume
   * OnTurnEnd: if still active when expiring, deal 10 self-damage

4. **Loan Shark**

   * OnPlay: `Money += BI * L` (ex L=1.0)
   * Add debt marker with duration 2 (stacks = # loans)
   * OnTurnEnd: when debt about to expire, require repayment with interest:

     * repay = round(BI * L * (1+r) * stacks)
     * if can’t pay: convert missing to HP loss
     * 5% catastrophe: lose 20% money + extra HP penalty

5. **Stocks & Bonds**

   * OnPlay add “STOCKS_BONDS” stack
   * OnRoundEnd: per stack, apply `Money += round(Money * rand(-0.2, +0.2))`

### System upgrade needed for Phase 1

`EffectRunner` must support more triggers:

* `OnTurnEnd`
* `OnDamageTaken`
* (optional but clean) `OnBeforeDealDamage`

And the engine must call them at the right times.
