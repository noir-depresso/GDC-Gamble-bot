# Commenting Guidelines

Use comments to explain intent, rules, and tricky flow consistently across the project.

- Add a short file or class comment when a file owns an important system boundary.
- Add method comments when the method represents orchestration, validation, or a gameplay rule boundary.
- Add block comments for important calculations, state transitions, or safety checks.
- Prefer explaining why a rule exists instead of re-stating the line of code.
- Keep wording plain and consistent across engine, bot, persistence, and effect files.
- When updating core files later, preserve the same comment style instead of mixing styles.
